Know Yourself AI – A Personal Tool for Clarity

By Ward Hasan · July 2025

This wasn’t just a weekend project. It was born out of a feeling — a weird mix of being overwhelmed by thoughts and wanting to sit with them. So, I started building a tool I personally needed: Know Yourself AI.

A lightweight, no-login, no-judgment assistant that lets you talk to yourself — with a little help from GPT.

🎯 The Idea

I didn’t want a chatbot that gives life advice. I wanted a mirror — something that reflects back your own thoughts, nudges you, and helps you connect dots you didn’t realize were scattered.

So, I wrote a system message: You are a thoughtful, reflective assistant... — and that’s where it began.

⚙️ The Tech Stack

  • Frontend: HTML, CSS, Vanilla JS
  • Backend: FastAPI + Uvicorn
  • Security Layer: A custom-built proxy API using FastAPI
  • AI: OpenAI’s GPT-4 API
  • Rate Limiting: slowapi (FastAPI wrapper)
  • Deployment: GitHub Pages (Frontend) + Render (Proxy & Backend)

🛡️ Security First

The biggest challenge wasn’t the prompt. It was protecting the key.

We decided early on that the OpenAI API key should never touch the frontend. Instead, we built a secure FastAPI backend and placed a proxy API layer between the frontend and the OpenAI service.

Here’s what we did:

  • ✅ Disabled /docs and /openapi endpoints
  • ✅ Added per-IP rate limiting (5 requests/min)
  • ✅ Used a X-Frontend-Secret header to validate the source of requests
  • ✅ Prevented direct access to backend — only the proxy can talk to it
  • ✅ Custom 404s for anyone poking around

🌍 Deploying with Peace of Mind

The frontend lives on GitHub Pages, styled with simplicity. The backend and proxy are both hosted on Render using their free tier (shoutout to them for making security-friendly deployments that simple).

Oh, and no frameworks on the frontend. We wanted everything to feel native, personal, and light.

💡 Lessons Learned

  • 🚧 Rate limits are not optional. Even for toy projects.
  • 🔐 Never expose secrets — not even once.
  • 🧠 Reflection is tech-enabled, but it has to be designed right. Prompt engineering is UX too.
  • 📦 A little proxy goes a long way.

This wasn’t just a tool. It was a mental health project disguised as a coding sprint.

Know Yourself AI drops tomorrow. Quietly. Without a launch party. But with a lot of heart.

If you ever needed a moment to sit with your thoughts, maybe this can help.
